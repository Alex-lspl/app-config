# AWG WARP — Android-приложение

Android-клиент AmneziaWG с автоматическим получением VPN-конфигурации через удалённый API,
без необходимости вручную копировать или вставлять файлы конфигурации.

Бэкенд-генератор конфигов: [warp-config-generator]

---

## Что делает приложение

Приложение — обёртка над протоколом AmneziaWG (форк WireGuard с обфускацией трафика),
которая умеет:

- подключаться к VPN по уже сохранённой конфигурации одним нажатием;
- автоматически получать новую конфигурацию с сервера без ручного ввода ключей;
- работать в фоне и поддерживать VPN-туннель активным;
- выбирать какие приложения должны идти через VPN, а какие — в обход (split tunneling);
- вести лог подключений для диагностики.

---

## Как работает получение конфигурации

```
Приложение                  Сервер (Vercel)              Cloudflare WARP
    │                              │                              │
    │  POST /api/generate          │                              │
    │  X-API-Key: ...              │                              │
    │ ──────────────────────────>  │                              │
    │                              │  Генерация X25519 ключей     │
    │                              │ ──────────────────────────>  │
    │                              │  Регистрация клиента         │
    │                              │ <──────────────────────────  │
    │                              │  peer_pub + IP адрес         │
    │  { config: "[Interface]..."} │                              │
    │ <──────────────────────────  │                              │
    │                              │                              │
    │  Сохраняет .conf локально    │                              │
    │  Запускает AmneziaWG туннель │                              │
```

1. Пользователь нажимает **«Создать конфигурацию»** в меню приложения.
2. Приложение отправляет запрос на сервер с выбранным регионом.
3. Сервер генерирует новую пару ключей WireGuard, регистрирует клиента в Cloudflare WARP
   и возвращает готовый `.conf` файл с параметрами обфускации AmneziaWG.
4. Приложение сохраняет конфигурацию и заменяет предыдущую (если была).
5. Пользователь подключается к VPN стандартным способом.

## Меню приложения

| Пункт меню                 | Действие                                                            |
|----------------------------|---------------------------------------------------------------------|
| Открыть конфигурацию       | Импорт `.conf` файла вручную с устройства                           |
| **Создать конфигурацию**   | Автоматическая генерация новой конфигурации через API               |
| Удалить конфигурацию       | Удаляет текущую сохранённую конфигурацию                            |
| Выбор приложений           | Настройка split tunneling — какие приложения используют VPN         |
| Расширенные настройки      | Выбор региона сервера (см. ограничение ниже)                        |
| Логи                       | Журнал подключений для диагностики проблем                          |

## Про выбор региона

Cloudflare WARP — anycast-сеть: физический сервер к которому подключается клиент
определяется автоматически по близости к пользователю, а не выбранным регионом
в конфиге. Поэтому смена региона в приложении **не меняет видимую геолокацию
(выходной IP)** — это ограничение самого протокола WARP в его бесплатной версии.

Выбор региона оставлен в **Расширенных настройках** как экспериментальная опция,
по умолчанию используется регион **«Стандартный»**.

---

## Защита от злоупотреблений

Сервер ограничивает частоту генерации конфигов на одно устройство:

- не более **1 генерации в минуту**;
- при **5 генерациях за 6 минут** — временная блокировка на **15 минут**.

Если лимит превышен, приложение покажет точное время до следующей попытки.

## Технологии

- **Kotlin** + **Jetpack Compose** — UI
- **AmneziaWG** (форк WireGuard) — VPN-туннелирование с обфускацией
- **HttpURLConnection** — сетевые запросы к API (без сторонних библиотек)
- **org.json** — парсинг ответов сервера
- **SharedPreferences** — хранение конфигурации и настроек пользователя

---

## Сборка проекта

1. Открой проект в Android Studio
2. В `VpnViewModel.kt` укажи адрес своего бэкенда:
   ```kotlin
   private const val API_BASE_URL = "https://твой-проект.vercel.app"
   private const val API_KEY = "твой_APP_API_KEY"
   ```
   (значения берутся из переменных окружения бэкенда — см. [warp-config-generator])
3. Собери APK через **Build → Build Bundle(s) / APK(s) → Build APK(s)**
